## 1. Core Mathematical Foundation (core/turbo_math.py)
- [x] **Random Orthogonal Rotation:** Implement a seeded Householder Transformation to rotate input vectors. This must be reversible (invertible) using the same seed.
      - Uses QR decomposition of seeded random matrix; inverse error ~9.54e-07 (machine precision)
- [x] **PolarQuant Stage:** Map the rotated Cartesian coordinates to Polar coordinates (Radius/Angles).
      - Computes L2 norms + normalized directions; quantizes norms with Lloyd-Max, directions uniformly
- [x] **Optimal Codebooks:** Implement a Lloyd-Max quantizer specifically tuned for the Beta distribution (the result of the rotation).
      - Iterative Lloyd-Max for half-normal distribution; 3-bit levels (8 centroids)
- [x] **QJL Correction:** Implement the 1-bit Quantized Johnson-Lindenstrauss residual corrector (storing only the sign +1/-1).
      - Random ±1 projections, packbits output, reconstruction via pseudo-inverse

## 2. Image Expert: "Turbo-Vision"

      - rotate(y, seed) → polar_quantize_encode → qjl_project on residual

      - norm_bits=3, dir_bits=3, n_projections=128
- [x] **Header:** Ensure the `rotation_seed` is saved per image to allow for perfect inverse rotation during unpacking.
      - seed stored in binary: [B turbo_flag][I rotation_seed][I polar_len][*][I qjl_len][*]
      - VERSION bumped to 5
